‚ùì Frequently Asked Questions

How accurate is the people counting? ‚ñº

The system achieves 90-95% accuracy in optimal conditions. Accuracy depends on:

  • ‚úÖ Camera angle (bird's eye view = best)
  • ‚úÖ Lighting conditions (bright = better)
  • ‚úÖ Video quality (1080p+ = best)
  • ‚úÖ Crowd density (fewer people = more accurate)
Why are predictions showing 0? ‚ñº

Solution: The system needs 10-15 seconds of data before it can make predictions.

  1. Wait at least 10 seconds after starting
  2. Check browser console (F12) for JavaScript errors
  3. Verify crowd_predictor.py file exists
  4. Refresh the page
What do the risk levels mean? ‚ñº
Risk Level Density Action Required
üü¢ LOW < 1 person/m¬≤ Normal operations
üü° MEDIUM 1-2.5 person/m¬≤ Increased monitoring
üü† HIGH 2.5-4 person/m¬≤ Take crowd control action NOW
üî¥ CRITICAL 4-5.5 person/m¬≤ Immediate intervention required
üî¥ EMERGENCY > 5.5 person/m¬≤ Evacuate immediately!
Can I use a live camera instead of video file? ‚ñº

Yes! You can use:

  • Webcam: Set VIDEO_PATH = 0
  • IP Camera: Set VIDEO_PATH = 'rtsp://camera-ip/stream'

Edit in app.py around line 15-20.

How do I change the physical area? ‚ñº

Method 1: Environment Variable (Recommended)

set PHYSICAL_AREA_M2=150 (Windows)
export PHYSICAL_AREA_M2=150 (Linux/Mac)

Method 2: Edit Code

In app.py, line ~15:

‚ö†Ô∏è Incorrect area = wrong density = wrong risk levels!
The count seems wrong - too high or too low. How to fix? ‚ñº

If count is TOO HIGH:

Edit yolo_detector.py, increase confidence:

self.conf_threshold = 0.02 # Increase from 0.001

If count is TOO LOW:

Edit yolo_detector.py, decrease confidence:

self.conf_threshold = 0.001 # Very sensitive
System is slow/laggy. How to improve performance? ‚ñº

Edit app.py for better performance:

FRAME_SKIP = 4 # Process every 4th frame (faster)
RESIZE_SCALE = 0.5 # Use half resolution
üí° This trades some accuracy for speed. Good for real-time monitoring on slower computers.
Can I monitor multiple cameras? ‚ñº

Current version supports 1 camera per instance. For multiple cameras:

  1. Run separate instances on different ports
  2. Example: Camera 1 on port 5000, Camera 2 on port 5001
  3. Edit app.py to change port number

üéØ Features Guide

üîÆ AI Predictions

The prediction system uses Double Exponential Smoothing to forecast future crowd levels.

How it works:
  • Analyzes past 100 data points
  • Detects trends (increasing/decreasing/stable)
  • Predicts 5, 10, 15 seconds ahead
  • Confidence score based on data stability

‚ö†Ô∏è Risk Assessment

Risk levels are calculated based on crowd density (people per square meter):

Density = People Count √∑ Physical Area (m¬≤)

üåä Surge Detection

Surge probability measures how rapidly the crowd is changing:

  • 0-30%: Normal, gradual changes
  • 30-60%: Moderate surge, monitor closely
  • 60-100%: High surge, take preventive action

üìä Density Meter

The circular meter shows crowd density as a percentage:

  • üü¢ 0-40%: Safe, comfortable spacing
  • üü° 40-70%: Crowded but manageable
  • üî¥ 70-100%: Dangerously crowded

üîß Troubleshooting Guide

Problem: System Won't Start

Error: ModuleNotFoundError: No module named 'cv2'

Solution:

pip install opencv-python numpy flask

Problem: Dashboard Shows "OFFLINE"

Solutions:

  1. Check if python app.py is running
  2. Refresh browser page (Ctrl+F5)
  3. Try URL: http://127.0.0.1:5000
  4. Check firewall settings

Problem: Video Feed Not Showing

Checklist:

  • ‚úÖ Verify video file exists
  • ‚úÖ Check video format (MP4, AVI supported)
  • ‚úÖ Test with: python -c "import cv2; cap=cv2.VideoCapture('video.mp4'); print(cap.isOpened())"
  • ‚úÖ Check file permissions

Problem: Predictions Always 0

Steps to fix:

  1. Wait 10+ seconds after starting (system needs data)
  2. Open browser console (F12), check for errors
  3. Verify crowd_predictor.py exists in project folder
  4. Restart application: Stop (Ctrl+C) and run python app.py again

Problem: High CPU Usage

Optimization:

# Edit app.py
FRAME_SKIP = 4 # Process every 4th frame
RESIZE_SCALE = 0.5 # Use half resolution
